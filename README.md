# Analise_BI_Microvarejo_dados

Diagnóstico de Dados no Microvarejo: Um Framework de Data Thinking

##  Sobre o Projeto
Este projeto foi desenvolvido com base em desafios reais enfrentados por pequenas empresas com baixa maturidade analítica. O objetivo foi transformar um cenário de dados fragmentados e processos informais em uma **estrutura de inteligência de negócios** capaz de orientar decisões estratégicas.

Utilizando a metodologia de **Data Thinking**, foram simulados os principais gargalos operacionais de um microvarejo para demonstrar como o rigor analítico pode identificar perdas e oportunidades de crescimento.

---

##  Desafios Abordados (Dores do Negócio)
Muitas empresas em estágio inicial sofrem com a ausência de uma cultura *Data-Driven*. Neste estudo, o foco foi mitigar:
*   **Inconsistência de Taxonomia:** Erros de cadastro que distorcem a visão de estoque.
*   **Ruído Financeiro:** Identificação de misturas entre fluxos de caixa PF (Pessoa Física) e PJ (Pessoa Jurídica).
*   **Capacidade Ociosa:** Análise de capital parado e produtos com baixo giro (Estoque Morto).
*   **Gaps de Processo:** Falta de padronização no uso de sistemas ERP.

---

##  Tecnologias e Metodologias
*   **Linguagem:** Python (Pandas, Numpy, Matplotlib, Seaborn).
*   **Estatística e ML:** Clusterização K-Means (Segmentação RFM) e Testes de Hipótese (Estatística Inferencial).
*   **BI & Analytics:** Criação de Dashboards de Diagnóstico, Curva ABC e Análise de Sazonalidade.
*   **Pair Programming:** Uso estratégico de IAs (Claude/Gemini) para otimização de lógica e exploração de cenários.

---

##  Resultados e Insights Principais
O projeto resultou em um **Dashboard Analítico** que permite ao gestor visualizar:
1.  **Curva ABC (Pareto):** Identificação de que 20% dos produtos geram 80% da receita.
2.  **Análise de Margem por Categoria:** Revelando quais setores sustentam o lucro da operação.
3.  **Investigação de Churn:** Estudo sobre o impacto da ruptura de estoque na fidelidade do cliente.
